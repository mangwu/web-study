Introduce:JS的错误检测

1.错误类型

代码错误分为两种：

  • 语法错误：存在拼写错误
  • 逻辑错误 ：执行结果与预期相悖，存在逻辑错误

2.修复语法错误

检查错误：

  1. 打开开发工具JavaScript控制台(shift + ctrl + i)， 在"Console"界面会直接返回一些浏览器检查到的错误
  2. 浏览器检查到的错误是很容易跟踪的：
    • 在火狐浏览器中红色的!表示一个错误;
      在谷歌浏览器中则是红色的╳
    • TypeError表示问题出现在哪儿
    • JavaScript 文件名，点击将跳转到开发者工具的“调试器”标签页。 如果你按照这个链接，你会看到错误突出显示的确切行。
    • 出错的行，以及导致错误的首个字符号。会在右边提示
    • 找到行号后检查语法是否有错误，编写是否正确
  3. 定向错误后，在修改JS源代码，再次用浏览器开发者工具检查

3.逻辑错误

逻辑错误和你要实现的功能有关，需要通过实际实现的功能，检查发现逻辑错误

4.其他常见错误

对于漏写等疏忽的错误，有一些很常见的错误提示：

  • SyntaxError: missing;before statement

    语法错误：语句缺少分号。

  • SyntaxError: missing ) after argument list

    语法错误: 参数末尾缺少括号。通常为函数/方法调用后结束括号没了

  • SyntaxError: expected expression, got 'string'

    语法错误：得到一个"string"而非表达式。字符串两端漏写了
