2.SVG

  • HTML中有一些元素能够创建简单图形，如<circle>和<rect>

<svg>的使用

  • svg元素有结束标签，其中内容为对图形的描述
  • version属性：1.1,版本号
  • baseProfile属性:用来供其他类型的验证识别SVG的版本
  • xmlns属性:正确的绑定命名空间:"http://www.w3.org/2000/svg"
  • width,height:容留svg图形的宽度和高度
  • svg中的内容可以有：
    • <rect>元素，width和height属性表示矩形的长宽，fill属性表示填充的颜色,x，y属性表示矩形的左上角的坐标。
    • <circle>,cx属性，cy属性表示圆心，r表示半径，fill表示填充颜色
    • 点击查看更多SVG元素
3.引用svg格式矢量图
  • 使用<img>,用法和位图一样。

    不一样的地方在于，对于不兼容svg的浏览器，提供了srcset属性：

    <img src="equilateral.png" alt="something" srcset="equilateral.svg">

    支持svg则引用srcset属性指引的svg内容，不支持就内嵌src指向的png资源

    优缺点：

    • 快速
    • 容易建成超链接
    • 无法使用javascript操作图像
    • CSS无法控制依靠<img>嵌入的SVG
    • 不能用CSS伪类来重设图像样式（如:focus）
  • 使用CSS引入

    本篇的背景就是CSS代码引入的：和img一样，也有兼容性问题，使用backgroud表示位图地址，使用 backgroud-image表示svg地址，优先选择backgroud-image地址，不支持svg再启用background

    使用 CSS 背景图像插入SVG 意味着它不能被 JavaScript 操作，并且也受到相同的 CSS 限制。

  • <SVG>元素内容引用

    使用阅读文本方式打开svg文件，复制svg代码到<svg>标签内。这种方法称为SVG内联

    优点

    • 减少引入资源的HTTP请求，减少加载时间
    • 可以为SVG分配class和id，使用CSS修改样式
    • 内联SVG是唯一可以让您在SVG图像上使用CSS交互（如:focus）和CSS动画的方法（即使在常规样式表中）。
    • 可建立超链接

    缺点

    • 多次使用会导致资源密集型维护
    • SVG代码增加HTML大小
    • 浏览器不能像缓存图片一样缓存内联SVG
  • <iframe>引入SVG

    不是个好办法,缺点

    • <iframe>有回退机制，不支持iframe会显示iframe中间的内容（回退内容）
    • 除非 SVG 和您当前的网页具有相同的 origin，否则你不能在主页面上使用 JavaScript 来操纵 SVG。
