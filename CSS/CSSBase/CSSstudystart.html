3.类名选择器

为元素添加类名（class）属性,给一类元素进行样式化

用法："." + 类名 {}

例子：下面列表中有两个有类名，只对它们进行样式化

  • li1
  • li2
  • li3
  • 这里包含一个和li2同类名的<span>元素
  • 这里包含一个和li2同类名的

    <p>元素

在上述例子中，li元素和span元素是同一样式，使用了同一类名选择器（.classname{})

但是p元素却和它们不一样，因为p元素使用了元素+类名选择器的模式，单独拿出来渲染， 它的意思是选中每个classname类的p元素（p.classname{})
一般不会这样使用，因为class的出现本来就是为了渲染一些（类）元素，而非把注意力放在元素上

4.包含选择器

HTML文档中的元素有包含和被包含关系，当需求是只想要某类元素内的一种元素有一种样式，而
不想要该元素外同一种元素有这种样式时，可以使用包含选择器。

例子：

我不想这个包含在<p>元素中的<em>元素有独特样式

我想让下面这个表格中的em元素有独特样式

  • 项目1
  • 项目2
  • 项目3

在表格中的em元素有了独特样式，而外面的em元素没有，使用的方法是：li em{}
使用空格表示二者元素之间的包含关系，这就是包含选择器。
"{}"作用的是后者，前者是为了让浏览器更快解析出需要渲染的元素

为了让浏览器更快的找到需要被渲染的元素，使用 相邻选择符(+)也是一种好选择

例如，渲染<h4>元素后的一个p元素，可以用这种方式：h4 + p {}

5.根据标签状态确定样式

元素有不同的状态，比如一个链接，有如下状态：

  • 未访问的,link
  • 访问过的,visited
  • 被鼠标悬停的,hover
  • 被键盘定位的
  • 被点击当中的

CSS可以对这些在不同状态下的元素进行修饰，
具体方法是，先定位元素，再使用":"（冒号），指明状态（visted等）,使用{}修饰

例如：未访问前为粉色，访问后为绿色,鼠标悬停是无下划线 

                    a:link {
                        color: pink;
                    }
                        
                    a:visited {
                        color: green;
                    }
                    a:hover {
                        text-decoration: none;
                    }
