CSS：层叠样式表

理解层叠(Cascading)的意思很重要

 将一些样式的效果作用于同一个元素，这种作用的方式就是层叠的（cascade）。
 有时候因为样式效果冲突，会使得期望的效果没有显示，当发生冲突时， 选择那条规则就是渲染样式效果的另外重要概念——specificity
 与期望效果不同，还有另外一种可能，就是css属性的继承性，有些属性继承，有些不继承 ，这会使得样式效果与预期不一样

1.层叠

 层叠：css规则的顺序很重要；当应用两条同级别的规则到一个元素时，写在后面的就是实际使用的规则 这就是层叠的实用含义。

2.优先级

 优先级是决定当多个规则有不同选择器对应相同的元素的时候，需要使用那个规则。

 可以这样理解，对于同一元素，可以使用不同的选择器选择（即使该选择器选择了多个元素，这一元素只是其中 之一），不同选择器中相同的属性会根据选择器的具体程度来决定谁被使用。 具体规则如下：

  • 一个元素选择器不是很具体-会选择页面上该类型的所有元素-优先级低
  • 一个元素选择器稍微具体一点-会选择页面中特定clas属性值或id值的元素-优先级高一些

 例如font-family和color会被子元素继承，除非在子元素上在在声明其他属性值的同类属性

  2. 控制继承

    CSS为每个属性提供了4个特殊的通用关于继承特性的属性值：

    1. inherit 设置该属性会使子元素属性和父元素相同。实际上，就是 "开启继承".
    2. initial 该属性被设置为和浏览器默认样式的相同值。浏览器默认样式未设置且该属性是自然继承的，就会设置为inherit
    3. unset 将属性重置为自然值，自然继承就是inherit。否则就是initial
    例子：
    • 默认链接
    • inherit链接
    • initial链接
    • unset链接

    分析上述例子：

    1. ul列表使用color属性声明颜色为red，是整体颜色
    2. 默认链接a元素没有添加额外样式，是a元素在浏览器中显示的color默认样式（蓝色）
    3. inherit链接选择a元素为color添加开启继承的属性值inherit，继承父元素颜色（red）
    4. initial链接选择a元素为color添加属性值initial，使color和浏览器默认color属性值相同（balck）
    5. unset链接选择a元素为color添加属性值unset，因为color属性是自然继承的，所以就是inherit(red)

    全体控制
    CSS中有一个属性可以代表其他所有属性表征它们的继承性。这个属性就是： all

    简单的说：
    • all可以使用inherit，initial，unset，revert中的任意一个值
    • all属性将这些值的继承特性应用到同一选择器中的所有属性
    • 使用all属性可以重置（unset）或者说是撤销对元素的所有样式更改，便于回到已知起点

4.理解层叠

需要理解的一个重要问题：
如何定义层叠在不止一个元素的时候怎么应用到CSS规则上：

  1. 重要程度
  2. 优先级
  3. 资源顺序

①资源顺序

对于同种规则，层叠具有覆盖性，最后面的规则会被应用。

②优先级

id选择器>class选择器>element选择器
需要注意：优先级是对比相同属性的在不同选择器下的优先性，不会覆盖其他所有规则
利用这种优先级规则，可以避免重复CSS，例子：

我是一个类的通用样式。
我也有通用样式，但是还应用了自己独特样式。small
我也有通用样式，但是还应用了自己独特样式。bright
我也有通用样式，但是还应用了自己独特样式。big

优先级的计算：不同选择器有不同分值，分值相加得到选择器权重，就可以进行匹配

  1. 千位：内联样式，声明在元素属性style内的属性得1分，无选择器，总分1000分
  2. 百位：选择器中包含ID选择器则得1分
  3. 十位：选择器中包含类选择器，属性选择器，或者伪类选择器得一分
  4. 个位：元素选择器，伪元素选择器得一分
注: 通用选择器 (*)，组合符 (+, >, ~, ' ')，和否定伪类 (:not) 不会影响优先级。

③重要程度

这个特殊的CSS可以覆盖上面所有css规则，用于修改特定属性的值

!important
例子：

我是有类名一段

我不仅仅有类名，还有id

都没有边框，但是第二段是利用id选择器设置了边框的，因为在为第一段添加样式时有这样一句：
border:none !important; 所以覆盖了所有类，都没有边框

使用顺序规则可以用另一个!important覆盖前面的，建议不要使用!important
