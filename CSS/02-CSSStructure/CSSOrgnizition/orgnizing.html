CSS代码组织

1.代码整洁

代码整洁对维护大型CSS代码很重要,保持代码整洁的方法：

  • 代码风格统一：统一使用两个空格缩进;统一命名方式;统一使用一种颜色描述方式
  • 代码可读性强：每个属性值新一行;每个选择器新一行;每个规则集隔一行
  • 添加合适注释：使用/* 注释 */添加注释

下面是一个很好的代码整洁的例子 

            .box {
              background-color: #567895; /* fallback for older browsers that don't support gradients */
              background-image: linear-gradient(to right, #ff0000, #aa0000);
            }

            p,
            article {
              font-size: 28px;
              color: white;
            }

2.逻辑结构

给HTML中的一般元素加上样式是一个很好的做法，一般来说这就是第一段逻辑段落，给这些元素加上规则：

  • body
  • p
  • h1 h2 h3 h4 h5
  • ul ol
  • table
  • a(链接)

下面是一个很好的第一段逻辑段落的样板： 

              /* || GENERAL STYLES */
              body {
                ……
              }

              h1,
              h2,
              h3 {
                ……
              }

              ul {
                ……
              }

              blockquote {
                ……
              }

第二段逻辑段落

如果要移除默认样式，需要一些实用类，这些是在很多不同元素运用的样式 

              /* || UTILITIES 实用类 */
              
              .nobullets {
                list-style: none
                margin: 0;
                padding: 0;
              }

              ……

第三段逻辑段落

整个网站都会用的东西：基础布局、抬头、或者导航页 

              /* || SITEWIDE */

              .mai-nav {

              }

              .logo {

              }

最后逻辑段

特指的东西，分为上下文，页面，组件等 

              /* || STORE PAGES */

              .product-listing {

              }

              .product-box {

              }

3.样式表分装

样式表过大不好维护，可以将它分成小样式表。可以分出一个普世样式表，需要特定规则的样式表等。

一个页面可以链接多个样式表，考前的样式表规则比后面的更有优先级

4.CSS的构建体系

程式化编写CSS，可以使用预处理和后处理工具。 预处理工具就是以一种编程语言进行网页设计，然后经过编译转发为正常的CSS样式表。

最流行的预处理工具是：Sass。

在Sass中，它能

  • 定义变量
  • 编译组件样式表
  • 后处理优化

定义变量

CSS原生功能可以自定义属性，使用Sass可以设置一个变量重复使用，修改时只需修改变量即可 

              $base-color: #123456;

              .alert {
                border: 1px solid $base-color;
              }

编译完成之后，得到如下CSS： 

              .alert {
                border:1 px solid #123456;
              }

编译组件样式表

我在上面提到了，一种组织CSS的方式是将样式表分成小的样式表。 在使用Sass时，你可以在另一个层次上理解，然后得到许多小样式表——甚至到了每个组件都有一个独立样式表的地步。 使用Sass中的include功能，这些都可以被编译为一个、或者少数几个真正链接到你的网站的样式表。
