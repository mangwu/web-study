盒模型

1.盒子

块级盒子与内联盒子

元素在网页中的表现方式有两种：block box 和 inline box； 两种盒子在页面流和元素之间关系表现不同行为：

  • 块级盒子：block
    • 在内联方向上扩展并占据父容器在该方向上的所有可用空间，意味着盒子和父元素一样宽
    • 盒子换行
    • weight和height有作用
    • 内边距padding和外边距margin和边框border会将其他元素从当前盒子周围“推开”
  • 内联盒子：inline
    • 盒子不会换行
    • weight和height不起作用
    • 垂直方向上的padding，margin和边框会应用但不会挤压其他inline盒子
    • 水平方向上的padding，margin和边框会应用会挤压其他inline盒子

上述的inline和block是相对于外部其他元素自身的显示特性。对于盒模型内部的显示类型有默认的方式: 正常文档流：意味着内部各个元素的显示类型如同inline，block一样。

可以对内部显示模型进行修改，如display属性的flex属性值， 它表明了一种新的显示类型：

3.CSS盒模型

完整的CSS盒模型应用于块级盒子，内联盒子只使用盒模型中定义的部分内容

盒模型定义了盒的每个部分：

  • margin box
    盒子最外面区域，是盒子和其他元素之间的空白区域
  • border box
    盒子的边界，边框盒包裹内容和内边距
  • padding box
    盒子内部，内容外部的区域
  • content box
    盒子内的内容区域，用于显示内容，用width和height设置大小

下面是盒模型的形象图：

标准盒模型

标准盒模型中，设置的width和height不是标准盒的大小，而是给content box设置的大小。 实际大小为：

  • a-height = width + padding-top + padding-bottem + border-top + border-bottem
  • a-width = height +padding-right + padding-left + border-left + border-right

注：margin box不算入实际大小中，margin会影响盒子在页面占据的空间，但是这是盒子的外部空间

替代盒模型

替代盒模型中，设置的width和height就是不包括margin box的和大小， 该模型宽度就是可见宽度，内容宽度需要减去padding和border宽度

浏览器默认使用标准模型，可以使用box-sizing:border-box设置为替代盒模型

}

4.margin，padding，border

外边距，内边距，边框，属于简写，可以直接设置盒子的四个边。

外边距

margin:盒子周围看不到的一圈空间，外边距不算在两种模型中的实际大小，外边距会影响和其他盒子的位置

margin属于简写属性，它可以一次控制盒子的4个边距，详细属性如下：

  • margin-top
  • margin-right
  • margin-bottem
  • margin-left

外边距折叠

外边距折叠指得是多个盒子之间的外边距设置选择间距关系：即两个外边距相接的元素， 这两个元素外边距会合成一个外边距，这个外边距大小为二者之间的大值。

 并不是所有相邻元素的外边距都会折叠。下面是折叠的例子：

我是外边距为1em的盒子
我是外边距为2em的盒子
实际上下间距大小为2em

边框

边框和外边距一样，是简写属性，有四个边框即四个详细属性：

  • border-top
  • border-right
  • border-bottem
  • border-left

每个边框又可以设置三种样式：宽度，样式，颜色

  • border-width
  • border-style
  • border-color

可以对单个边框设置各个样式：宽度，样式，颜色

  • border-top-width
  • border-top-style
  • border-top-color
  • ……
