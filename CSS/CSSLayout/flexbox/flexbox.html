1.弹性盒子作用

CSS布局中长期跨浏览器兼容的布局技术是float和positioning；但是它们有一些局限性：

  • 在父内容里垂直居中一个块内容
  • 使容器的所有子项占用等量的可用宽度/高度
  • 使多列布局中所有列采取相同高度，即使包含的内容量不同

3.flex模型说明

flex模型沿两个轴来布局：

  • 主轴main axis, flex项的放置方向, 主轴的开始和借宿被称为main start, main end
  • 交叉轴 cross axis, 垂直于flex元素放置的轴, 交叉轴开始和结束被称为cross start 和cross end
  • display: flex的父元素被称为flex容器(container)
  • flex容器中的直接子元素被称为flex项(flex item)

4.主轴方向

flexbox提供了属性flex-drection指明主轴方向，默认值为row按照语言方向排列。 除此之外还有如下值:

  • column,列布局，从上到下排列
  • row-reverse,行布局，排列方向与语言方向相反
  • column-reverse,列布局，排列方向从下到上.如下

5.换行

如果flex项过多，不仅会占据满整行，而且会溢出导致超出容器，增加横向滚动条。 当flex项过多时，可以使用flex-wrap: wrap定义容器，让其直接子元素在溢出时自动换行。 同时可以对直接子元素使用flex: 300px;等对子元素宽度做出限制，意味着其宽度至少为300px；

本html文档页面布局使用了上述的属性，只是flex: 480px有所不同。

6.flex-flow缩写

应当知道flex-direction和flex-wrap有缩写flex-flow，表示弹性盒子文档流的基本属性。 横向流还是纵向流，自动换行或列还是不换行和列。一般而言是这样的：flex-flow: row wrap;

8.flex属性值

上述中flex的属性值有宽度最小值，无单位比例值等，它们是可以一起赋予flex使用的，如flex:2 840px; flex属性值有三项，可以简写：

  • 无单位比例，可以使用flex-grow单独指定
  • flex-shrink，也是无单位比例，用于溢出容器的flex项,取出一定溢出量防止溢出。高级弹性盒子，少用
  • flex-basis，宽度最小值。三个值一般不分开写，用flex已经足够了

9.对齐

上面所有例子中的flex项因为内容多，占据整个宽度，所以不需要特定布局。但是对于宽度大于内容宽度时， 就需要对flex项进行对齐处理。以下是一个未对齐例子:

使用align-items对flex项在交叉轴上对齐：

  • stretch,默认值，如上例子中所示，它会使flex项沿着交叉轴的方向拉伸以填充父元素， 如果父元素没有固定高度，就拉伸到同样的高度（与最高的flex项保持一致）
  • center,保持原有高度，但是会在交叉轴居中，设置按钮垂直居中很有用
  • flex-start,flex-end,使flex项在交叉轴的开始或结束位置对齐

align-self属性可以对单个flex项使用，覆盖父元素的align-item属性， 其属性值和align-item一样。

控制flex项在主轴位置的属性是justify-content，其属性值有:

  • flex-start,默认值，从主轴开始处排列
  • flex-end,让flex项右对齐（结尾处）
  • center,在主轴居中
  • space-around，沿主轴均匀分布，很有用
  • space-between,不在两端留下空间的均匀分布

上面的例子中,最后一个显示的和space-around一样，但是盒子模型不一样，margin:0 auto; 是直接对flex项进行的样式，盒子模型中的左右间距是中间的空白横向距离。而对容器使用justify-content: space-around 不会使容器的直接子元素的margin改变，而是会让容器处理其直接子元素的对齐方式

10.flex项排序

flexbox中有order属性可以改变flex项的布局顺序，但是不会影响到源顺序(dom树中的顺序)； 所有flex项的order属性默认值为0，其实现布局排序细节如下：

  • order值大的flex项显示顺序靠后
  • order值相同的flex项按照源顺序显示
  • order值可以是负数，让它比值为0的元素排列的靠前
